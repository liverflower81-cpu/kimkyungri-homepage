## 2. 내가 직접 해야 하는 작업 (브라우저 로그인이 필요해서 대신 할 수 없음)

### Step 1. Netlify에 사이트 배포 (실제 도메인 받기)

1. https://app.netlify.com 접속 → GitHub 계정으로 로그인
2. "Add new site" → "Import an existing project" 클릭
3. "Deploy with GitHub" 선택 → `kimkyungri-homepage` 저장소 선택
4. Build settings는 그대로 두고(별도 빌드 명령 없음, 정적 파일이라 비워둬도 됨) "Deploy site" 클릭
5. 배포 완료되면 `https://무작위이름.netlify.app` 형태의 주소가 생성됨
   → 이 주소가 앞으로 메타 연결에 사용할 "실제 도메인"

> 참고: 사이트 이름은 Netlify 대시보드 "Site settings → Change site name"에서
> `kimkyungri-homepage.netlify.app` 처럼 원하는 이름으로 바꿀 수 있음

### Step 2. 자동 배포 확인

- 이후로는 GitHub 저장소에 코드를 push할 때마다 Netlify가 자동으로 재배포함
- 즉, Claude가 `index.html`을 수정하고 git push만 하면 사이트가 자동 갱신됨

### Step 3. Meta(Facebook) 쪽 설정 — 내일 수업에서 이어서 할 부분

1. https://developers.facebook.com 접속 → 로그인
2. "내 앱" → 기존에 만들던 앱 선택 (또는 새 앱 생성)
3. "Facebook 로그인" 제품 설정 화면으로 이동
4. 아래 항목에 Step 1에서 받은 Netlify 주소를 입력:
   - **앱 도메인(App Domains)**: `무작위이름.netlify.app`
   - **웹사이트 URL**: `https://무작위이름.netlify.app`
   - **개인정보처리방침 URL**: `https://무작위이름.netlify.app/privacy.html`
5. 저장 후 "유효한 OAuth 리디렉션 URI"에도 같은 도메인 기반 주소 등록
6. (필요시) Meta 비즈니스 관리자에서 "도메인 인증"을 요구하면,
   `index.html`의 `<head>` 안 아래 주석 위치에 인증 메타태그를 추가:
   ```html
   <!-- 여기에 추후 Meta 도메인 인증 / Meta 픽셀 코드를 넣으면 됩니다. -->
   ```
